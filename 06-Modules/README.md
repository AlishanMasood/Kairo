# Modules

## Purpose

This folder contains documentation for individual modules, services, and components within the Kairo platform. Each module should have its own subfolder.

## What Belongs Here

- Module specifications
- API documentation for individual modules
- Module-level architecture details
- Configuration references
- Module dependency documentation
- Data models specific to a module

## What Does NOT Belong Here

- Cross-cutting architecture concerns (use `04-Architecture`)
- Product-level feature descriptions (use `02-Products`)
- Development process documentation (use `05-Development`)
- Release notes (use `07-Releases`)
