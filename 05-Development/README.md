# Development

## Purpose

This folder contains development guidelines, processes, workflows, and engineering standards for teams working on the Kairo platform.

## What Belongs Here

- Coding standards and style guides
- Git workflow and branching strategies
- CI/CD pipeline documentation
- Testing strategies and requirements
- Code review guidelines
- Environment setup instructions
- Debugging and troubleshooting guides

## What Does NOT Belong Here

- Architecture documentation (use `04-Architecture`)
- Product requirements (use `02-Products`)
- Governance policies (use `00-Governance`)
- Module-specific documentation (use `06-Modules`)
